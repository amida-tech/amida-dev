Master

The goal of working with PHIX is to assemble a master view of an individuals healthcare record. The master endpoints allow you interact with the healthcare record for a PHIX users, allowing you to get the master record, as well as add, update and work with individual elements of the users record.

  • /master (GET) - Returns the master record for the authenticated user.
  • /master/ (POST) - Catch all method used to update a specific element in the authenticated users record.
  • /master/ccda (POST) - Takes a JSON object used to filter the master record, and returns a master ccda.
  • /master/{username}/{element} (GET) - Retrieves a specific element in the currently authenticated users record.
  • /master/{username}/{element} (POST) - Updates a specific element in the currently authenticated users record.
  • /master/{username}/{element} (PUT) - Adds a specific element in the currently authenticated users record.

You can access, and play with all the master methods, using the PHIX interactive documentation.
