Access

Each individual PHIX user gets control over which providers have access to their personal health information exchange. All access endpoints provide you with the resources you need to make, accept and remove provider requests for access to individual PHIX user personal health information accounts.

  • /access/pending (GET) - Return a list of any pending provider request for the authenticated account.
  • /access/pending (POST) - Approves a pending provider request for access for the authenticated account.
  • /access/pending (DELETE) - Removes a pending provider request for access for the authenticated account.
  • /access (GET) - Returns a list of providers who have been granted access to the authenticated account.
  • /access (POST) - Updates the access for a provider who has been granted access to the authenticated account.
  • /access (DELETE) - Removes the access for a provider who has been granted access to the authenticated account.

You can access, and play with all the access methods, using the PHIX interactive documentation.
