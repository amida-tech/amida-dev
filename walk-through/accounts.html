Accounts

Each user of PHIX is assigned an account, which are available via the /account grouping. All account endpoints provide you with the resources you need to manage all users of a particular installed PHIX system.

There are 9 separate methods you can use to integrate with PHIX accounts:

  • /login (POST) - Initial login for all API integrations.
  • /loggedin (GET) - Checks to see if a user is logged in.
  • /verify (POST) - Checks to see if an individual is a verified PHIX user.
  • /validuser (POST) - Checks to see if a users is a valid user in the PHIX system.
  • /switch/:user (GET) - Switch from one user to another.
  • /account (GET) - Retrieves the account information for the currently logged in user.
  • /account (POST) - Updates the logged in users account information.
  • /account (PUT) - Adds a new user account into the PHIX system.
  • /logout (POST) - Loggs out the current existing users.

You can access, and play with all the account methods, using the PHIX interactive documentation.
