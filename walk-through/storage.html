Storage

In the PHIX system, an individuals healthcare information is stored in the PHIX filesystem. The storage group of endpoints gives you access to all the endpoints for managing storage files. Before healthcare records can be made an active part of an individuals record, the files need to be added to storage, parsed and made part of a users healthcare information store.

  • /storage (GET) - Gets a listing of all files stored for a specific user.
  • /storage (PUT) - Loads a record into the storage grid, JSON body 'file' contains file data
  • /storage (POST) - Allows parsed flag to be passed to trigger parsing of CCDA files.
  • /storage/record (GET) - Takes a JSON object containing file id and returns the file's contents in JSON.
  • /storage/record/{identifier} (GET) - Returns the object associated with the identifier as a file download.
  • /storage/{identifier} (DELETE) - Removes a record from the file store.
  • /storage/preview (POST) - Takes a JSON object of identifier and returns that record for preview.

You can access, and play with all the storage methods, using the PHIX interactive documentation.
